# M011 - Versioned Prompt System for ppt-agent

**Author:** Rebecca (Architect)
**Date:** 2026-03-11
**Branch:** `feature/m011-ppt-agent-versioned-prompts`
**Status:** Complete

## Summary

Added the versioned prompt system to ppt-agent, following create-agent's established pattern exactly. The agent previously had a single 1.0.0 prompt version with no versioning infrastructure.

## Changes

### `agents/ppt-agent/src/registry.ts`
- Added version 1.1.0 entry to `versions[]` array (identical config to 1.0.0 as baseline)
- Added `OfficeAgent.EnablePptAgent@1.1.0` flight for version switching
- Added descriptive comments matching create-agent's style
- Default remains 1.0.0 -- fully backward compatible

### `agents/ppt-agent/src/prompts/1.1.0/`
- Full copy of 1.0.0 as scaffold for future prompt iterations
- Includes system-prompt.md, rai.md, connected/interop system prompts, all skills, and scripts

### `agents/ppt-agent/CLAUDE.md`
- New developer reference documenting prompt versions, flight flags, and how to add future versions/overrides

### No changes to `ppt-agent.ts`
- BaseAgent + ConfigResolver in `@officeagent/core` already handle version resolution, flight application, and overwrites via the registry config. No agent code changes needed.

## Design Decisions

1. **Exact pattern match with create-agent** -- used `versions[]` + `flights[]` with version override, same naming convention (`OfficeAgent.EnablePptAgent@1.1.0`)
2. **1.1.0 is content-identical to 1.0.0** -- this is infrastructure only; prompt content changes come separately
3. **Preserved existing search skills flight** -- `OfficeAgent.EnablePptAgentSearchSkills` unchanged

## Activation

```env
OAGENT_FLIGHTS=OfficeAgent.EnablePptAgent@1.1.0:true
```
