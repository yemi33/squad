## Gotchas
- **EPERM on yarn install in worktrees**: If node_modules already exist from a previous partial install, Windows may lock files. `Remove-Item -Recurse -Force node_modules` may or may not help. Best to use a fresh worktree or an existing one with clean node_modules.
- **Lage `start` vs Vite `dev`**: `yarn start --to bebop` runs the production server entry point. Always use `yarn dev` from `apps/bebop/` for local development.
