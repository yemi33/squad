## What was built

A CI eval regression gate that automatically blocks PR merges when agent quality scores drop below configurable baseline thresholds.

### Components
1. **Gate script** (`eval/ci_eval_gate.py`) -- runs eval aliases, extracts scores from reports, compares against baselines with tolerance, exits non-zero on regression
2. **Baseline config** (`eval/ci_eval_baseline.json`) -- defines CI aliases to run, per-dimension thresholds (societas metrics), tolerance percentages, and absolute min floors
3. **Pipeline integration** -- `eval_gate` job added to `build-container-pr.yml`, runs after build+tests, publishes gate report artifact
4. **Unit tests** (`eval/tests/test_ci_eval_gate.py`) -- covers threshold logic, score extraction, baseline loading, summary formatting
5. **Documentation** (`docs/CI_EVAL_GATE.md`) -- operation guide, threshold update procedures, emergency override instructions

### Design choices
- **Dual threshold model**: baseline + tolerance% regression check AND absolute min_score floor per dimension
- **Three exit codes**: 0=pass, 1=fail, 2=skipped (distinct for CI auditing)
- **Report-only mode**: can check thresholds against existing reports without re-running evals
- **Emergency override**: `OAGENT_SKIP_EVAL_GATE=true` env var or `--skip` flag
- **Fast CI subset**: starts with `ppt.example` (single PPT + societas eval, <30 min)

### Known limitations
- Baseline values are estimates; need calibration against actual ppt.example scores on main
- eval_gate job rebuilds from scratch (no artifact sharing between pipeline jobs)
- Single alias coverage (PPT only) -- designed to grow over time via ci_aliases array
