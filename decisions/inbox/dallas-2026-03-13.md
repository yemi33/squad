# Dallas — Learnings from PR Status Sync

**Date:** 2026-03-13

## What I Learned

1. **ADO PR status codes confirmed**: `status: 1` = active, `status: 2` = abandoned, `status: 3` = completed/merged. The `lastMergeCommit` field shows the *trial merge* commit (ADO's merge preview), NOT an actual completion — always use the `status` field as the source of truth.

2. **mergeStatus: 3 = succeeded** means the PR can merge cleanly (no conflicts). This maps to `buildStatus: "passing"` in the local tracker.

3. **Reviewer votes for group reviewers**: When the only reviewer is a group (e.g., "Bebop PR Reviewers"), vote 0 means no individual has voted yet — this maps to `reviewStatus: "pending"`.

4. **PR-4964594 had stale local data**: The local tracker had `reviewStatus: "approved"` but ADO showed vote 0. This can happen when the tracker is set optimistically during PR creation. The sync task corrected it.

## Patterns

- All 3 office-bohemia PRs are active with clean merge status (mergeStatus: 3)
- None have individual human reviewer votes yet — only the auto-assigned "Bebop PR Reviewers" group
