# Dallas — Learnings from PR Status Sync

**Date:** 2026-03-13

## Key Learnings

1. **Azure CLI (`az`) is broken on this DevBox** — DLL load failure for `win32file` in the `portalocker` module. All `az` commands fail, including `az repos pr show`. This affects both bash and PowerShell invocations.

2. **Git credential manager is the reliable auth source** — Use `git credential fill` with `protocol=https` and `host=office.visualstudio.com` to get a bearer token (JWT). This works from bash but NOT from Node.js `execSync` (heredoc syntax issue on Windows).

3. **ADO REST API PR status mapping:**
   - `status: "active"` = PR is open (maps to squad status `"active"`)
   - `mergeStatus: "succeeded"` = no merge conflicts, build passing -> `buildStatus: "passing"`
   - Reviewer `vote: 0` with at least one reviewer -> `reviewStatus: "pending"`

4. **ADO PR reviewers are team groups, not individuals** — The office-bohemia repo auto-assigns team groups like `[TEAM FOUNDATION]\Bebop PR Reviewers` and `[TEAM FOUNDATION]\Bebop Bundle Size Approvers`.

## Gotchas

- **Never use `az` CLI on this machine** until the win32file DLL issue is resolved. Use `curl` + bearer token from git credential manager instead.
- **Node.js v24 on Windows** escapes `!==` in `-e` eval strings. Always write to a `.js` file instead of using inline eval.
- **heredoc (`<<<`) doesn't work in Windows cmd** — only works in bash/git-bash shell.
