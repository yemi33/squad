## What I Learned

1. **Both office-bohemia PRs are still active with no reviewer votes** — PR-4959092 (cowork route) and PR-4959405 (W003 design doc) both have `status: 1` (active), `mergeStatus: 3` (merge succeeded/no conflicts), and only the "Bebop PR Reviewers" group assigned with vote 0 (no vote).

2. **ADO `lastMergeCommit` is a test merge, not actual merge** — The merge commit shown in ADO PR responses (with comment "Merge pull request NNNN...") is ADO's automatic test merge to check mergeability. The actual PR status field (`status: 1/2/3`) is the source of truth for whether a PR is open, abandoned, or completed.

3. **No individual human reviewers assigned** — Both PRs only have the "Bebop PR Reviewers" team group as reviewer. No individual votes have been cast.

## Patterns

- PR sync is straightforward: fetch via `mcp__azure-ado__repo_get_pull_request_by_id`, map status/mergeStatus/reviewer votes, compare with local tracker.
- When all reviewer votes are 0 and there's at least one reviewer → `reviewStatus: "pending"`.
- `mergeStatus: 3` → `buildStatus: "passing"` (merge check succeeded).
